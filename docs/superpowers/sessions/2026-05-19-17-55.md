## Key Discoveries
- `CropWaterRate` seeding: must call `SeedCropsAsync()` BEFORE `SeedCalculatorReferenceDataAsync()` — crops must exist before rates can reference them.
- Global Restrict override in ApplicationDBContext.OnModelCreating overrides any `.OnDelete(SetNull)` on CropWaterRate.IrrigationSystemId — omit the OnDelete call, let the global loop handle it.
- Razor RZ1031: `@(condition ? "selected" : "")` on `<option>` inside `<select name=...>` triggers tag helper conflict. Use `selected="@(condition)"` instead.
- `dotnet test --no-build` after `dotnet ef migrations add` will fail with PendingModelChangesWarning if the test DLL wasn't rebuilt. Always `dotnet build` first.
- FieldAndCrop has no explicit CropId/IrrigationSystemId FK properties — EF uses shadow properties. Access via `.Include(...).Crop.CropId`.
