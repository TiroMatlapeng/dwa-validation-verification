# UI Re-skin + Labels Implementation Plan (Plan 2 of 4)

**Goal:** Land a DWS-branded token system (`dws.css`), make `_Layout` role-aware with a login shell, and apply `[Display(Name=...)]` to the user-visible properties on the main entities — so screens read in English and share one brand.

**Architecture:** The existing `_Layout.cshtml` already has a sidebar + main shell. Rather than rewrite it, we augment: add `dws.css` with tokens + component classes used by the Account/Users views, extend the sidebar to reflect signed-in role (hide admin entries from non-admins, show user name + Logout), add `_LoginLayout.cshtml` for auth screens, and sweep `[Display]` attributes.

**Tech Stack:** Razor / vanilla CSS custom properties. No framework.

**Source of truth for colours:** `docs/wireframes/modern-internal-wireframes.html` (hex values extracted).

---

## Execution (inline — no subagent dispatch)

Each step is a single commit with `Co-Authored-By` trailer.

### Step 1 — Brand tokens and component CSS

**File:** `wwwroot/css/dws.css` (new)

Contents: CSS custom properties for DWS palette + type scale + spacing + radii, then component classes referenced by Plan 1's views: `.dws-page-header`, `.dws-card`, `.dws-card--login`, `.dws-panel`, `.dws-form`, `.dws-form-row`, `.dws-label`, `.dws-input`, `.dws-checkbox`, `.dws-validation-summary`, `.dws-field-error`, `.dws-table`, `.dws-btn`, `.dws-btn-primary`, `.dws-btn-secondary`, `.dws-btn-ghost`, `.dws-btn-danger`, `.dws-status-pill`, `.dws-status-pill--ok`, `.dws-status-pill--warning`, `.dws-inline-form`.

Commit: `Add dws.css brand tokens and component classes`

### Step 2 — Link dws.css and make sidebar role-aware

**File:** `Views/Shared/_Layout.cshtml`

- Add `<link rel="stylesheet" href="~/css/dws.css" asp-append-version="true" />` after `site.css`.
- Gate the "Admin" sidebar group (`Owner` and new `Admin/Users` link) behind `@if (User.IsInRole("SystemAdmin"))`.
- Replace the hard-coded "TM" avatar with `@User.FindFirst("displayName")?.Value` + their role, plus a Logout form-POST button.
- Add an "Admin / Users" sidebar link under System when admin.

Commit: `Make _Layout role-aware and surface user + logout`

### Step 3 — `_LoginLayout.cshtml` for Account views

**File:** `Views/Shared/_LoginLayout.cshtml` (new)

A minimal shell (no sidebar, centered card container, DWS brand colour background). Switch `Views/Account/Login.cshtml` and `AccessDenied.cshtml` to use `_LoginLayout`.

Commit: `Add _LoginLayout for anonymous auth screens`

### Step 4 — `[Display(Name = ...)]` sweep on key entities

**Files:** model class edits only.

Apply `[Display]` to visible properties of:
- `Models/ApplicationUser.cs` — `FirstName`, `LastName`, `EmployeeNumber`, `IsActive`, `OrgUnitId`.
- `Models/Property.cs` — `SGCode`, `PropertyReferenceNumber`, `QuaternaryDrainage`, `WmaId`, `CatchmentAreaId`, `PropertySize`, `ProclamationDate`, `RegistrationDate`, `Longitude`, `Latitude`, `PropertyStatus`.
- `Models/FileMaster.cs` — `RegistrationNumber`, `CaseNumber`, `FileNumber`, `SurveyorGeneralCode`, `PrimaryCatchment`, `QuaternaryCatchment`, `FarmName`, `FarmNumber`, `RegistrationDivision`, `FarmPortion`, `WarmsApplicant`, `Notes`, `AssessmentTrack`.
- `Models/OrganisationalUnit.cs` — `Name`, `Type`, `ProvinceId`, `WmaId`, `CatchmentAreaId`.
- `Models/WaterManagementArea.cs` — `WmaName`, `WmaCode`.
- `Models/Province.cs` — `ProvinceName`, `ProvinceCode`.
- `Models/CatchmentArea.cs` — `CatchmentCode`, `CatchmentName`, `WmaId`.

Convention: Title Case English, abbreviations stay capitalised (`SG Code`, `WARMS Registration Number`, `SAPWAT`, `WMA`, `CMA`).

Commit: `Apply [Display] attributes to user-visible entity properties`

### Step 5 — Final verification + merge
