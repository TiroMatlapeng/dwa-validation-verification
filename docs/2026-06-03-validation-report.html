🔴 Critical — ship-blocker (1)

DOC-01 — Statutory letter PDFs are publicly accessible

Signed Section 35 / 33 letters are written to wwwroot/_uploads/letters/ and served by static-file middleware with no authentication, no scope check, no virus gate.

Confirmed live: an anonymous request — curl -kL http://localhost:4000/_uploads/letters/VV-9476BEAF-001-S35_L1.pdf — returned HTTP 200, Content-Type: application/pdf, a valid 38 KB %PDF. The reference format VV-{8hex}-{NNN}-{code} is structured/guessable.

Impact: exposure of personal, legally sensitive ELU correspondence — a POPIA data-protection breach. Irony: case documents are already correctly gated through a controller + database-blob storage; only letters bypass it.

Fix: serve letters via an authenticated, scope-checked controller action (mirror /Document/Download) and move the blob root out of wwwroot.

Status: pre-existing (Letters feature). Confirmed by both the code review and the live run.

Suggested remediation plan

  1. Now / blocker: DOC-01 — move letters out of wwwroot; serve only via a scope-checked controller action.
  2. Security hardening: AUTH-01 (catchment/province scoping + cache key), DOC-02 (virus-scan gate + magic bytes), X-04 (patch packages), DOC-03.
  3. Concurrency correctness: WF-01 (RowVersion + transaction on transitions), WF-02 (unique index + collision-safe reference numbers + transactional issuance).
  4. Functional bugs: BUG-030 (Owner access for NationalManager), river-seed de-duplication.
  5. Test infrastructure: X-01 — add SQL Server-backed integration tests so reporting/guard regressions can't slip past InMemory.
  6. Scheduled scope (not ship-blockers): report filter UI, dam CalculatorEngine, and the remaining Minor cleanups.
