1. Executive Summary

This document provides two infrastructure sizing tiers for an on-premise deployment of the DWA Validation & Verification (V&V) system at the DWS Head Office data centre in Pretoria, with connectivity to 9 provincial offices and multiple regional/CMA offices.

CONSERVATIVE Minimum Viable

Sized for the confirmed user base, moderate concurrency, and essential redundancy. Suitable for initial rollout with room for controlled growth.

LIBERAL Production Recommended

Sized for full national rollout, peak concurrency, document-heavy workflows, high availability, and 5-year growth without re-procurement.

Key driver: The system manages approximately 500,000 property records across 19 Water Management Areas with ~200–400 concurrent internal users and an external public portal serving water users, landowners, and legal representatives.

Bandwidth Calculation — Head Office Internet

Formula: Peak users × average page weight × pages per minute × overhead factor
Conservative: 150 users × 250 KB × 3 pages/min × 1.3 overhead = ~14 MB/min = ~2 Mbps sustained
With document uploads: Add 20 concurrent 5 MB uploads/min = ~15 Mbps burst
Public portal peak: 500 external users at 2 pages/min = ~5 Mbps additional
Total peak: ~22 Mbps — well within 100 Mbps conservative allocation, with headroom for growth.

Why These Numbers?

Per-user bandwidth requirement:
A typical V&V workflow session involves:
  • Page navigation (HTML + assets): ~250 KB per page, ~3 pages/min = ~100 Kbps per user
  • Document upload (title deed, SG diagram): 5 MB upload every ~5 min = ~130 Kbps per user
  • Document download/viewing: 3 MB download every ~3 min = ~130 Kbps per user
  • Total per active user: ~360 Kbps (0.36 Mbps)
Conservative (15 concurrent users): 15 × 0.36 = 5.4 Mbps sustained → 10 Mbps link provides ~85% headroom for bursts.
Liberal (40 concurrent users): 40 × 0.36 = 14.4 Mbps sustained → 50 Mbps link provides ~70% headroom.

9. Network & Bandwidth — Regional / CMA Offices

Regional and CMA offices are smaller sites with 3–10 V&V users. Many are in rural areas with limited connectivity options.

ParameterCONSERVATIVELIBERAL

Expected V&V users3–55–10
Peak concurrent2–35–8
Latency tolerance< 120 ms< 60 ms
Fallback connectivity4G/LTE mobile broadbandDual WAN (primary fibre + 4G/LTE failover)
Rural connectivity challenge: Some CMA offices in remote areas (e.g., Mzimvubu, Vaal) may only have 4G/LTE connectivity (typically 5–20 Mbps with variable latency of 30–100 ms). The application must be optimised for higher latency: minimise round trips, use pagination, compress responses, and avoid large inline assets. Consider allowing offline document preparation with batch upload capability for the most remote sites.

15. Recommendation

RECOMMENDED Phased Approach

Start with Conservative infrastructure for the initial 2–3 WMA pilot, then scale to Liberal specifications before full national rollout. This:

  • Reduces upfront capital expenditure
  • Validates actual usage patterns before committing to HA infrastructure
  • Allows procurement of the Enterprise SQL Server license only when Always On AG is truly needed
  • Provides real performance baselines for right-sizing the production environment

Critical Path Items

  • WAN connectivity to provincial offices is the most likely bottleneck — engage SITA or DWS IT infrastructure team early to provision MPLS/SD-WAN links
  • SQL Server licensing is the single largest cost item — evaluate Standard vs Enterprise based on actual HA requirements
  • Document storage growth is the primary scaling concern — plan for expandable SAN/NAS from the start
  • Rural CMA connectivity requires application-level optimisation (pagination, compression, offline capability)
